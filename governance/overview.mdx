Vexel operates without executives or central authority — only code and community. Every authenticated user is a Sovereign node with the power to shape the ecosystem by voting on active governance proposals. This section explains how the DAO works, what you see on each proposal card, and how The Mesh keeps governance state synchronized across the network.

## Accessing governance

Navigate to **Governance** in the sidebar to open the governance panel. The terminal fetches all active proposals automatically.

<Note>
  Governance is a protected section. If you are not authenticated, the terminal redirects you to the Sovereign Link screen. Complete Sovereign Link authentication before proceeding.
</Note>

## What proposals are

Proposals are suggested changes to the Vexel system submitted for community review. A proposal can cover:

- New platform features
- Fee structure adjustments
- Protocol upgrades

Any change that affects the Vexel ecosystem must pass through the proposal and voting process before it takes effect.

## Proposal cards

Each active proposal is displayed as a card in the governance grid. Every card contains the following fields:

<CardGroup cols={2}>
  <Card title="Title and description" icon="file-lines">
    A short title and a plain-language description of the proposed change and its intended effect.
  </Card>
  <Card title="Neural Consensus insight" icon="brain">
    An AI-generated analysis of the theoretical impact the proposal would have on the ecosystem, shown in the Neural Consensus section of the card.
  </Card>
  <Card title="meshSentiment badge" icon="signal">
    A live badge showing the current community sentiment signal from The Mesh (for example, "Analyzing..." while data is being gathered).
  </Card>
  <Card title="Vote buttons" icon="square-check">
    Two action buttons — **Authorize FOR** and **Authorize AGAINST** — let you cast your vote directly from the card.
  </Card>
</CardGroup>

## Neural Consensus

Before you vote, each proposal card surfaces a **Neural Consensus** insight — an AI analysis of the theoretical impact the proposal would have on the Vexel ecosystem. This analysis appears in a highlighted block on the card beneath the proposal description.

<Info>
  Neural Consensus is informational only. The AI analysis does not influence, weight, or override your vote. You make the final decision.
</Info>

## How proposals are fetched

The governance panel calls `GET /api/market/public/proposals` to retrieve the list of active proposals. The endpoint is publicly readable, so you can browse proposals without an active session — but casting a vote requires Sovereign Link authentication.

## The Mesh and governance synchronization

The Mesh is the decentralized infrastructure that keeps Vexel running across distributed nodes. Every 15 seconds, The Mesh emits a pulse that synchronizes all governance states globally. When you cast a vote, it is propagated across the network through this mechanism, ensuring your participation is recorded consistently across all nodes.

<Tip>
  The meshSentiment badge on each proposal card reflects live data from The Mesh. Check it before voting to understand the current community leaning on a proposal.
</Tip>
