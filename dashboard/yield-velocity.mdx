Yield Velocity is Vexel Horizon's real-time capital growth metric. Rather than showing a static annual percentage yield, the terminal calculates and displays exactly how much value your portfolio accumulates every second — and that number changes continuously as market conditions shift. It is the clearest answer to the question: "How fast is my money working right now?"

## How it differs from APY

Traditional finance expresses returns as an annualized figure (APY) that is recalculated periodically and displayed as a static number until the next update cycle. Yield Velocity abandons that model entirely.

<CardGroup cols={2}>
  <Card title="Static APY">
    Calculated once per period. Displayed as a fixed percentage. Does not reflect intra-day movements in fees, market price, or governance rewards.
  </Card>
  <Card title="Yield Velocity">
    Recalculated on every Mesh pulse (~every 15 seconds). Displayed as a dollar-per-second figure with four decimal places. Reflects real-time asset performance, network fees, and active governance incentives.
  </Card>
</CardGroup>

## How it is calculated

The **Adaptive Engine** drives Yield Velocity. On each Mesh pulse it evaluates three inputs:

<Steps>
  <Step title="Asset performance">
    The current market price and unrealized gain or loss for each asset in your portfolio contribute proportionally to the overall velocity figure.
  </Step>
  <Step title="Network-generated fees">
    Protocol fees generated by activity on the Mesh flow back to Sovereign nodes. The Adaptive Engine includes your share of these fees in the per-second calculation.
  </Step>
  <Step title="Governance incentives">
    Active participation in DAO governance — voting on proposals, submitting strategies — generates additional incentive rewards that the engine folds into the velocity figure.
  </Step>
</Steps>

The result is a single growth figure expressed as dollars per second, shown with four decimal places:

```
+$0.0240/sec
```

This value is delivered to the dashboard inside each Mesh Pulse update, and the display refreshes automatically — you don't need to reload the page to see the latest figure.

## How it is displayed

The UI counter updates on two different cycles to keep the display smooth.

**On each Mesh pulse (~15 seconds):** The authoritative value from the server replaces the local estimate. The displayed figure snaps to the true server-calculated velocity.

**Every 100 ms between pulses:** The terminal applies linear interpolation to the running balance, advancing it smoothly toward the next expected value. This prevents the counter from appearing frozen between server updates.

<Info>
  The interpolation is a display smoothing technique. The canonical net worth value is always set by the server on the next Mesh pulse and overrides any accumulated client-side estimate.
</Info>

## Collective vs. Sovereign velocity

The label and data source for Yield Velocity change depending on your session state.

| Mode | Label | Source |
|------|-------|--------|
| Collective (guest) | Collective Velocity | `GET /api/market/public/yield/status` |
| Sovereign (authenticated) | Personal Velocity | `GET /api/market/yield/personal` |

Both endpoints return a `velocity_per_second` value that the dashboard reads on load and displays as the opening figure for the counter.

## Increasing your Yield Velocity

<Tip>
  Higher allocations to Real World Assets (RWA) and active participation in Governance proposals are the two most direct levers for increasing your personal Yield Velocity. RWA positions generate protocol fees, and governance activity unlocks incentive rewards — both of which the Adaptive Engine folds into your per-second figure.
</Tip>
