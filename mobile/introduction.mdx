Vexel Mobile, known as **The Guardian**, is the primary user interface and biometric authentication layer of the Vexel ecosystem. It serves as your personal portal for managing AI-driven dynamic portfolios, executing transactions, and authorizing sessions on other Vexel surfaces such as Vexel Horizon.

Built with Flutter and powered by Riverpod, the app is designed around a simple principle: **no private keys, no sensitive AI logic, and no write authorization ever happens on any device other than your phone.**

## The Guardian principle

Vexel Mobile is the only device that can commit capital in the Vexel ecosystem. Every write action — a portfolio rebalance, an AI proposal approval, a DAO vote triggered from Horizon — arrives at the phone as a `pending_action` event over Socket.io and is displayed inside the `ActionApprovalDialog`. The user reviews the details, presses **Sign and Authorize**, authenticates via FaceID, TouchID, or fingerprint, and the app relays the signature to Vexel Core via `POST /auth/authorize`.

This design keeps the attack surface on mobile minimal. The app never stores private keys and never runs sensitive AI logic locally — Vexel Core handles inference and cryptographic validation, while the phone owns the one thing that cannot be phished remotely: the user's biometric intent.

## Architecture at a glance

Vexel Mobile uses a layered architecture with strong MVVM principles, powered by Riverpod for reactive state management.

| Layer | Location | Responsibility |
| :--- | :--- | :--- |
| Entry point | `lib/main.dart` | `ProviderScope`, theme wiring, `AuthWrapper`, WebSocket bootstrap |
| Core config | `lib/core/config/` | `AppConfig` — API and WebSocket URLs |
| Models | `lib/models/` | Data entities such as `MarketAsset` |
| Screens | `lib/screens/` | Presentation widgets for every route |
| Services | `lib/services/` | `AuthService`, `MarketService`, `BiometricController`, `LocalizationService`, `ThemeService`, `NfcTetheringService`, `UserService` |
| Widgets | `lib/widgets/` | Reusable UI such as `AssetCard`, `ActionApprovalDialog`, `VoiceCommandWidget` |
| Localization | `assets/lang/` | JSON translation files (`en.json`, `es.json`) |

See [Architecture](/mobile/architecture) for the full component diagram and data flow.

  Vexel Mobile communicates with the `Vexel-Core` backend over HTTPS and Socket.io. All endpoints are derived from `AppConfig` (`https://vexel.es/api` by default). See [API reference](/mobile/api) for the complete list.
</Note>
