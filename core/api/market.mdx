The Market module is the largest surface in Vexel Core. It aggregates quotes and fundamentals from multiple providers, drives watchlists and narrative portfolios, routes trades, generates reports, and exposes public ecosystem metrics for Guest Mode in Horizon.

## Base path

All endpoints are rooted at `/market`. Marketplace and webhook routes use `/marketplace` and `/webhooks` respectively.

## Authenticated endpoints

The following endpoints require a bearer JWT.

### Dashboards and preferences

| Method | Endpoint | Purpose |
|---|---|---|
| `GET` | `/market/dashboard` | Personalized dashboard for the authenticated user |
| `GET` | `/market/correlations` | Portfolio correlation matrix |
| `GET` | `/market/risk-scan` | Risk scan across holdings (via Brain) |
| `GET` | `/market/stress-test` | Stress test against the current portfolio (via Brain) |

### Watchlists

| Method | Endpoint | Purpose |
|---|---|---|
| `GET` | `/market/watchlist` | Read the user's watchlist |
| `POST` | `/market/watchlist` | Add an asset to the watchlist |
| `DELETE` | `/market/watchlist` | Remove an asset from the watchlist |

### Narrative portfolios

| Method | Endpoint | Purpose |
|---|---|---|
| `GET` | `/market/portfolios` | List the user's narrative portfolios |
| `POST` | `/market/portfolios` | Create or update a narrative portfolio |
| `POST` | `/market/strategy/canvas` | Save a Neural Canvas strategy and simulate (via Brain) |

### Trading and execution

| Method | Endpoint | Purpose |
|---|---|---|
| `GET` | `/market/trading/balance` | Trading account balance |
| `POST` | `/market/trading/order` | Place a buy or sell order |
| `GET` | `/market/trading/history` | Trading history |
| `POST` | `/market/execute` | Initiate a sensitive operation (see approval flow) |
| `POST` | `/market/authorize` | Authorize a pending operation with a Mobile signature |
| `POST` | `/market/voice/command` | Process a voice command (via Brain) |
| `POST` | `/market/autonomous/rebalance` | Start an autonomous rebalance |

<Warning>
  Operations initiated via `/market/execute`, `/market/voice/command`, and `/market/autonomous/rebalance` do **not** commit immediately. They are parked and require `/market/authorize` with a valid Mobile signature. See [Real-time events](/core/api/realtime) for the approval flow.
</Warning>

## Response shape

Most endpoints return JSON with the following envelope:

```json
{
  "status": "ok",
  "data": { }
}
```

Error responses use NestJS defaults:

```json
{
  "statusCode": 401,
  "message": "Unauthorized",
  "error": "Unauthorized"
}
```

## Data providers

Vexel Core composes data from Finnhub, CoinGecko, Alpha Vantage, and Twelve Data. Provider failures are tolerated through fallback logic — a missing fundamentals provider, for example, does not break a dashboard response.

<Info>
  History endpoints read from and write to TimescaleDB. `HistorySyncService` runs a daily sync at midnight that persists watchlist and benchmark history and refreshes continuous aggregate views for fast range queries. You can also trigger a sync on demand with `POST /market/sync-history`.
</Info>
