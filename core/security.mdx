---
title: "Security and ledger"
description: "How Vexel Core protects payloads, verifies signatures, encrypts secrets, and maintains an immutable audit trail."
---

Vexel Core applies defense-in-depth across every sensitive path. Payloads between services are signed, user secrets are encrypted at rest, signatures are verified through an HSM-compatible interface, and every critical action is recorded to an append-only ledger.

## Service-to-service signing

`SecurityService` signs every outbound HTTP request from Core to Vexel Brain using a shared secret (`SHARED_SECRET`). Brain verifies the signature before processing the request, blocking spoofed calls that bypass Core.

The same service exposes `GET /market/security/quantum-check`, a public endpoint that generates a sample hybrid post-quantum signature for the current payload.

## Quantum-hybrid signatures

`HsmService` produces and verifies hybrid signatures that combine classical elliptic curve cryptography with a post-quantum scheme seeded from `PQ_MASTER_SECRET`. Sensitive mobile approvals can include both a classical `signature` and a `pqcSignature`. Core verifies both before executing.

<Info>
  Hybrid signatures let Vexel maintain compatibility with existing secure elements on mobile devices while buying protection against harvest-now-decrypt-later attacks on classical keys.
</Info>

## Encrypted vault

`VaultService` wraps the application secret store. It uses `VAULT_MASTER_KEY` (a 32-character key) to encrypt values before writing them and decrypts on read. Use the vault for any credential that has to round-trip through the database.

## Trust scoring

`TrustService` computes reputation and trust metrics used across the Marketplace controller — for example, when ranking public narrative portfolios. Scores combine historical performance, voting activity, and adherence to governance outcomes.

## Redis mesh

`MeshService` encapsulates the Redis pub/sub channel that Core uses to coordinate background jobs across horizontally-scaled instances. The mesh is how a single broadcast (`broadcastPulse`, `marketAlert`) reaches every Socket.io connection regardless of which process owns the client.

## Immutable ledger

`LedgerService` writes every critical action — executions, authorizations, governance votes, session links — to an append-only JSONL file. Each entry includes the hash of the previous entry, forming a tamper-evident chain.

Monitor integrity with the public endpoint:

```bash
curl http://localhost:3000/market/ledger/integrity
```

A truthy response confirms the chain is intact. Integrate this call into a health check or monitoring pipeline to raise alerts when an entry has been altered or removed.

<Warning>
  The ledger file is a local artifact by default. For production deployments, back it up to cold storage and mirror it into a replicated object store so that a compromised host cannot destroy the audit trail.
</Warning>
