Vexel Horizon keeps every surface live through a single Socket.io connection. The browser opens a WebSocket to `window.location.origin` on load, registers its session, and then receives a continuous stream of events that update the dashboard, market ticker, execution feedback, and Community Board. This page catalogs every event the client sends and receives.

## Connection

The terminal connects to `window.location.origin` on load and is typically reverse-proxied by Nginx to Vexel Core. There are no additional query parameters on the initial handshake. Authentication is layered on top of the connection by emitting `registerSession` immediately after `connect`.

## Client-to-server events

### `registerSession`

Registers a new browser session with Vexel Core so the server can route targeted events (for example, `sessionLinked` after a successful Sovereign Link scan).

| Argument | Type | Description |
|---|---|---|
| `sessionId` | `string` | Unique session identifier generated by `crypto.randomUUID()` on the client. |

```javascript
socket.on('connect', () => {
  socket.emit('registerSession', sessionId);
});
```

## Server-to-client events

### `sessionLinked`

Emitted when Vexel Mobile scans the Sovereign Link QR and signs the session. The terminal stores the returned JWT and user profile, adds `is-sovereign` to the `<body>` element, and reloads the dashboard with personal data.

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "name": "Ada Sovereign",
    "email": "ada@example.com"
  }
}
```

### `marketPulse`

Emitted approximately every 15 seconds. Drives the Yield Velocity counter, the Market Intelligence chart, and the header ticker. Between pulses, the client interpolates `totalValue` every 100 ms using `yieldVelocity`.

```json
{
  "totalValue": 1245680.0,
  "yieldVelocity": 0.024,
  "prices": {
    "BTC": { "price": 64231.42, "change": 2.4 },
    "ETH": { "price": 3450.18, "change": -1.1 },
    "AAPL": { "price": 172.5, "change": 0.8 },
    "PAXG": { "price": 2341.0, "change": 1.2 }
  },
  "globalSentiment": 72.5
}
```

### `action_completed`

Emitted after a Sovereign action resolves on the execution engine — for example, a swap. The client renders the result in the execution feedback panel and appends an entry to the AI feed. When `quantum_secured` is `true`, the badge on the feedback panel reads **Quantum Secured** to indicate the action was verified by the lattice proof flow.

```json
{
  "result": {
    "id": "swap_a83f",
    "reasoning": "Executed 0.5 BTC → 8.2 ETH at 3450.18",
    "qty": 0.5,
    "symbol": "BTC"
  },
  "quantum_secured": true
}
```

### `session_revoked`

Emitted when the user revokes the browser session from Vexel Mobile. The client clears `vexel_jwt` and `vexel_user`, logs a `SECURITY` entry to the feed, and force-reloads into Guest Mode after a brief delay.

```json
{
  "reason": "Revoked from mobile device.",
  "revokedAt": "2026-04-22T18:00:00Z"
}
```

<Warning>
  `session_revoked` can arrive at any time. Always treat the browser session as ephemeral — your JWT can be invalidated remotely from Vexel Mobile.
</Warning>

## Reconnection behavior

If the underlying socket drops, the client falls back to **Fail-Safe Sandbox Mode**. Cached data remains visible, but live updates pause until the Mesh connection is restored. On reconnect, the client re-emits `registerSession` with the same `sessionId` and resumes receiving `marketPulse` updates.
